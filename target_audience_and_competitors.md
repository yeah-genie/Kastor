# Kastor v2 타겟 사용자 및 경쟁사 분석

## 1. 타겟 사용자 (Target Audience)
**"데이터를 통해 의사결정을 내려야 하지만, 코딩이나 복잡한 툴 학습은 부담스러운 실무자"**

### A. 그로스 마케터 (Growth Marketer)
- **상황:** 광고 성과, 퍼널 전환율 등을 매일 분석해야 함.
- **니즈:** 엑셀보다 빠르고, 대시보드 툴보다 유연한 분석 환경. 데이터 소스(페이스북, GA4 등)를 쉽게 연결하고 싶어 함.

### B. 프로덕트 매니저 (Product Manager)
- **상황:** 제품 사용 로그를 분석하여 기능 개선점을 찾아야 함. SQL을 할 줄 알지만 시간이 오래 걸림.
- **니즈:** 빠르게 가설을 검증하고, 팀원들에게 근거 데이터를 시각화해서 보여주고 싶어 함.

### C. 초기 스타트업 창업자 (Founder)
- **상황:** 데이터 분석가가 없음. 투자자 보고용 지표(MRR, Churn Rate 등)를 직접 뽑아야 함.
- **니즈:** 비용 효율적이고, 전문 지식 없이도 "질문"만으로 인사이트를 얻고 싶어 함.

### D. 데이터 분석가 (Data Analyst) - *확장 타겟*
- **상황:** 반복되는 데이터 추출 요청에 지쳐있음. 간단한 분석은 자동화하고 싶어 함. 주니어 분석가의 경우 복잡한 환경 설정 없이 바로 분석하고 싶어 함.
- **니즈:** 
    - **효율성:** 코딩보다 빠르게 초안(Draft)을 잡고 싶음.
    - **Code Eject:** GUI로 만든 로직을 Python/SQL 코드로 내보내 심화 분석에 활용하고 싶음.
    - **Custom Logic:** 정해진 블록 외에 직접 코드를 삽입할 수 있는 유연성을 원함.

### E. 데이터 비전문가 (Non-Technical Users) - *New*
- **상황:** "카페 창업을 하고 싶은데..." 처럼 구체적인 데이터 분석 질문보다는 **추상적인 문제 해결**을 원함. 어떤 데이터를 봐야 하는지조차 모름.
- **Pain Point:**
    - **질문의 구체화 어려움:** "매출 추이 보여줘"가 아니라 "장사 잘 되게 해줘"라고 질문함.
    - **도구 사용 미숙:** 블록을 직접 추가하고 연결하는 것조차 부담스러움.
- **니즈:** "알아서 다 해주는(Agentic)" 경험. 파일만 던져주면 인사이트까지 자동으로 도출되길 원함.

## 2. 경쟁사 및 기존 대안의 Pain Point

### A. LLM 챗봇 (ChatGPT, Claude 등)
*   **Black Box (불투명성):** AI가 "매출이 30% 올랐습니다"라고 답변했을 때, **어떤 데이터와 로직으로 계산했는지 확인하기 어려움.** 신뢰성 문제 발생.
*   **휘발성 (Ephemerality):** 대화가 길어지면 맥락을 잃거나, 지난주에 분석했던 방식을 **똑같이 재사용하기 어려움.**
*   **수정의 어려움:** 중간 단계(예: 환율 적용 환산)만 살짝 바꾸고 싶어도, 프롬프트를 다시 처음부터 작성해야 하는 경우가 많음.

### B. 전통적 BI 툴 (Tableau, PowerBI)
*   **높은 진입 장벽:** 툴을 배우는 데만 수주가 걸림. 간단한 차트 하나 그리는데도 수많은 설정이 필요함.
*   **무거움:** 데이터 전처리(ETL) 과정이 복잡하여, 원본 데이터가 깔끔하지 않으면 시작조차 못함.

### C. AI 데이터 분석 툴 (Zapier AI, Julius AI 등)
*   **수동적인 도우미:** 사용자가 "차트 그려줘"라고 해야만 그려줌. 먼저 "이 데이터를 보니 이런 차트가 필요해 보입니다"라고 제안하는 **주도성(Proactivity) 부족**.
*   **선형적 구조의 한계:** 복잡한 로직을 채팅으로만 풀어나가면 전체 구조(Forest)를 보기 어려움.

## 3. Kastor v2의 해결 솔루션 및 구현 기준 (Why Kastor?)

### ✅ Flow-First, Chat-Assisted (흐름 중심, AI 보조)
*   **Pain Point 해결:** 챗봇의 **불투명성**과 **휘발성** 해결.
*   **솔루션:** 대화는 시작일 뿐, 결과물은 **"블록 스택(Block Stack)"**으로 남습니다. 사용자는 AI가 만든 로직을 시각적인 블록(Load -> Transform -> Visualize)으로 확인하고 검증할 수 있습니다.
*   **구현 기준:**
    1.  **Zero-Friction Transition:** 채팅 요청 시 캔버스에 **애니메이션**과 함께 블록이 생성되며, 시선(Viewport)이 자연스럽게 이동해야 함.
    2.  **Context-Aware Creation:** 생성된 블록은 이전 블록과 **자동으로 연결(Connect)**되어야 함 (데이터 스키마 자동 매핑).
    3.  **Chat-Block Sync:** 채팅 메시지 클릭 시 해당 블록이 하이라이트되어 역추적 가능해야 함.

### ✅ White-Box AI (설명 가능한 AI)
*   **Pain Point 해결:** 결과에 대한 **신뢰성** 문제 해결.
*   **솔루션:** 각 블록은 내부 로직(필터링 조건, 계산 수식 등)을 투명하게 보여줍니다(`Inspector Panel`). 사용자가 원하면 언제든 직접 개입하여 수정할 수 있습니다.
*   **구현 기준:**
    1.  **Code View Toggle:** 모든 블록의 우상단에 `< >` 버튼 제공, 클릭 시 **실제 실행 코드(Python/SQL)** 노출.
    2.  **Diff Highlighting:** AI 수정 요청 시(예: "날짜 포맷 변경"), 변경된 코드 부분을 `Diff View`로 비교하고 **[승인/거절]** 기능 제공.
    3.  **Step-by-Step Execution:** 블록 단위 실행 및 각 단계의 **중간 데이터 결과(Output Preview)** 즉시 확인 가능.

### ✅ Agentic Workflow Generation (능동적 워크플로우 생성) - *New*
*   **Pain Point 해결:** 추상적 질문을 가진 비전문가의 **진입 장벽** 및 **도구 사용 미숙** 해결.
*   **솔루션:** "카페 창업 성공하려면?" 같은 질문을 AI가 스스로 **[데이터 로드 -> 전처리 -> 시각화 -> 인사이트]** 단계로 분해하여 전체 파이프라인을 제안합니다.
*   **구현 기준:**
    1.  **Abstract Query Decomposition:** 추상적 질문을 구체적 분석 단계로 쪼개는 **CoT(Chain of Thought)** 과정을 UI로 시각화 (예: "Thinking... Analyzing Goal... Planning Workflow...").
    2.  **Proactive Plan Preview:** 바로 실행하지 않고, "이런 분석 흐름을 준비했습니다"라고 미리 보여준 뒤(Plan Preview), 사용자가 승인하면 **일괄 생성**.
    3.  **Context Persistence:** 랜딩 페이지에서 파일을 업로드하면, 워크스페이스 진입 즉시 해당 파일을 분석할 수 있는 `Load Block`을 자동으로 세팅.

### ✅ Infinite Canvas & Grouping (무한 캔버스 및 그룹화)
*   **Pain Point 해결:** 복잡한 분석 시 **구조 파악의 어려움** 해결.
*   **솔루션:** 줌/팬이 가능한 캔버스 위에서 자유롭게 블록을 배치하고, 관련된 작업끼리 **그룹화(C-Shape)**하여 정리할 수 있습니다. 스크래치처럼 직관적인 조작감을 제공합니다.
*   **구현 기준:**
    1.  **Semantic Grouping:** AI 요청("전처리 과정 묶어줘") 시 관련 블록(`Load`+`Transform`)을 자동 그룹화 및 네이밍.
    2.  **Smart Layout:** 복잡한 워크플로우를 자동 정렬해주는 **Auto-Layout** 기능.
    3.  **Navigational Minimap:** 미니맵 클릭 시 해당 위치로 점프하는 내비게이션 기능.

### ✅ Reusable Workflows (재사용성)
*   **Pain Point 해결:** 매번 처음부터 다시 해야 하는 **비효율** 해결.
*   **솔루션:** 완성된 블록 스택은 저장하여 언제든 새로운 데이터(예: 다음 달 매출 파일)만 갈아끼워 똑같은 분석을 수행할 수 있습니다.
*   **구현 기준:**
    1.  **Parameterization (변수화):** 특정 값(연도, 파일명 등)을 **변수(Variable)**로 지정하여 실행 시 입력받는 폼(Form) 생성.
    2.  **Template Gallery:** 워크플로우를 템플릿으로 저장하고 공유하는 라이브러리.
    3.  **Data Swap:** 드래그 앤 드롭으로 데이터 파일 교체 시 로직 자동 재실행.

## 4. 현재 MVP 구현 상태 평가 (Current MVP Status Evaluation)

### ✅ 달성 (Achieved)
- **Flow-First UI:** AI 채팅 패널에서 버튼 클릭으로 블록을 생성하고 워크스페이스에 배치하는 핵심 흐름 구현 완료.
- **Infinite Canvas & Grouping:** 줌/팬, 미니맵, C자형 그룹 블록(접기/펼치기 포함) 구현 완료.
- **Data Source:** 파일 업로드(CSV, Excel) 및 드래그 앤 드롭, Mock 클라우드 커넥터 구현 완료.
- **Block Management:** 다중 선택, 삭제, Undo/Redo, 블록 간 시각적 스냅(Snap) 구현 완료.
- **Chat-Only Initial State:** 초기 진입 시 채팅 중심의 경험 제공 후 워크스페이스로 자연스럽게 전환 구현 완료.
- **Analyst Tools:** Code View, Auto-Layout, Template Gallery, Execution Flow 구현 완료.

### ⚠️ 보완 필요 (Needs Improvement) - *Next Steps*
1.  **Agentic & CoT:**
    - 추상적 질문에 대한 자동화된 사고 과정(CoT) 시각화 및 다단계 블록 자동 생성 로직 필요.
2.  **Landing Context Fix:**
    - 랜딩 페이지 파일 업로드 시 워크스페이스에서 다시 묻지 않고 즉시 로드 블록 생성 필요.
